# Skill: db-migration-runner

Use this skill whenever you are modifying the Prisma schema or running database migrations for VerifyNG.

---

## Before You Start

1. Read `agents/rules/architecture.md` — Prisma lives at `server/prisma/`, not inside `src/`
2. Read `agents/rules/security.md` — understand which fields are sensitive and must be constrained at the schema level
3. Never edit any file inside `server/prisma/migrations/` manually

---

## Prisma Setup

### Location

```
server/
└── prisma/
    ├── schema.prisma     ← the only file you edit
    └── migrations/       ← auto-generated by Prisma — never touch manually
```

### Prisma Client Singleton

Create a shared Prisma client instance used across all services:

```typescript
// server/src/utils/prisma.ts

import { PrismaClient } from '@prisma/client';

const globalForPrisma = globalThis as unknown as { prisma: PrismaClient };

export const prisma =
  globalForPrisma.prisma ??
  new PrismaClient({
    log: process.env.NODE_ENV === 'development' ? ['query', 'error', 'warn'] : ['error'],
  });

if (process.env.NODE_ENV !== 'production') {
  globalForPrisma.prisma = prisma;
}
```

Import this in every service file — never instantiate `new PrismaClient()` anywhere else.

## Full Schema — VerifyNG

This is the complete `schema.prisma` for the VerifyNG MVP. Do not add models or fields beyond this without confirmation.

```prisma
// server/prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id           String    @id @default(cuid())
  email        String    @unique
  passwordHash String
  displayName  String
  role         UserRole  @default(BUYER)
  isVerified   Boolean   @default(false)
  createdAt    DateTime  @default(now())
  updatedAt    DateTime  @updatedAt

  reviews            Review[]
  reports            Report[]
  verificationTokens VerificationToken[]
  ownedVendors       Vendor[]            @relation("VendorOwner")
}

enum UserRole {
  BUYER
  VENDOR
  ADMIN
}

model Vendor {
  id                String              @id @default(cuid())
  businessName      String?
  instagramHandle   String?
  phoneNumber       String?
  ownerId           String?
  claimStatus       ClaimStatus         @default(UNCLAIMED)
  claimedAt         DateTime?
  state             String?
  city              String?
  category          String?
  description       String?
  whatsappUrl       String?
  tiktokUrl         String?
  facebookUrl       String?
  linkedinUrl       String?
  featured          Boolean             @default(false)
  trustScore        Float               @default(0)
  trustLabel        String              @default("Unrated")
  reviewCount       Int                 @default(0)
  scamFlag          Boolean             @default(false)
  moderationFlag    Boolean             @default(false)
  createdAt         DateTime            @default(now())
  updatedAt         DateTime            @updatedAt

  owner             User?               @relation("VendorOwner", fields: [ownerId], references: [id], onDelete: SetNull)
  reviews           Review[]
  summary           VendorSummary?
  reports           Report[]

  @@index([businessName])
  @@index([instagramHandle])
  @@index([phoneNumber])
  @@index([state])
  @@index([category])
  @@index([claimStatus])

  // Note: Add a GIN index for full-text search on businessName post-MVP
  // CREATE INDEX "Vendor_businessName_gin" ON "Vendor" USING gin(to_tsvector('english', "businessName"));
}

enum ClaimStatus {
  UNCLAIMED
  PENDING_APPROVAL
  CLAIMED
}

model Review {
  id                  String    @id @default(cuid())
  vendorId            String
  userId              String
  rating              Int
  reviewText          String
  transactionChannel  String?
  orderDate           DateTime?
  verifiedBuyer       Boolean   @default(false)
  isFlagged           Boolean   @default(false)
  createdAt           DateTime  @default(now())
  updatedAt           DateTime  @updatedAt

  vendor              Vendor    @relation(fields: [vendorId], references: [id], onDelete: Cascade)
  user                User      @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([vendorId, userId])
  @@index([vendorId])
  @@index([userId])
}

model VendorSummary {
  id                       String    @id @default(cuid())
  vendorId                 String    @unique
  summaryText              String?
  deliveryReliability      String?
  customerSatisfaction     String?
  recurringComplaints      Json      @default("[]")
  trustPatterns            String?
  scamReason               String?
  generatedAt              DateTime  @default(now())
  reviewCountAtGeneration  Int       @default(0)

  vendor                   Vendor    @relation(fields: [vendorId], references: [id], onDelete: Cascade)
}

model Report {
  id          String        @id @default(cuid())
  vendorId    String
  userId      String
  reason      String
  description String?
  status      ReportStatus  @default(PENDING)
  createdAt   DateTime      @default(now())

  vendor      Vendor        @relation(fields: [vendorId], references: [id], onDelete: Cascade)
  user        User          @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@index([vendorId])
  @@index([userId])
  @@index([status])
}

enum ReportStatus {
  PENDING
  REVIEWED
  DISMISSED
}

model VerificationToken {
  id        String                @id @default(cuid())
  userId    String
  token     String                @unique
  type      VerificationTokenType
  expiresAt DateTime
  createdAt DateTime              @default(now())

  user      User                  @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@index([token])
  @@index([userId])
}

enum VerificationTokenType {
  EMAIL_VERIFICATION
  PASSWORD_RESET
}

model ContactMessage {
  id        String                 @id @default(cuid())
  name      String
  email     String
  subject   String
  message   String
  status    ContactMessageStatus   @default(PENDING)
  createdAt DateTime               @default(now())
}

enum ContactMessageStatus {
  PENDING
  RESOLVED
}

model BlacklistedToken {
  id        String   @id @default(cuid())
  tokenHash String   @unique
  expiresAt DateTime
  createdAt DateTime @default(now())

  @@index([expiresAt])
}

model RateLimit {
  key       String   @id
  count     Int
  resetAt   DateTime
  createdAt DateTime @default(now())

  @@index([resetAt])
}
```

## Migration Commands

Run these from inside the `server/` directory:

```bash
# Create and apply a new migration (development)
npx prisma migrate dev --name describe_what_changed

# Apply pending migrations (production)
npx prisma migrate deploy

# Reset database and re-apply all migrations (development only — destroys data)
npx prisma migrate reset

# Generate Prisma client after schema changes
npx prisma generate

# Open Prisma Studio to inspect data
npx prisma studio
```

---

## Migration Rules

- Run `npx prisma migrate dev --name [description]` every time `schema.prisma` changes
- The migration name must describe the change clearly: `add_vendor_scam_flag`, `create_report_table`
- Never run `prisma migrate reset` in production — it deletes all data
- Always run `npx prisma generate` after schema changes to update the Prisma client types
- Commit both `schema.prisma` and the generated `migrations/` folder to version control
- Never manually edit SQL files inside `migrations/`

## Schema Constraints Reference

| Model | Field | Constraint | Reason |
|---|---|---|---|
| User | role | `UserRole` enum default BUYER | Only BUYER, VENDOR, ADMIN allowed |
| Vendor | ownerId | `@relation("VendorOwner")` onDelete: SetNull | Vendor unclaimed if user deleted |
| Vendor | claimStatus | `ClaimStatus` enum default UNCLAIMED | Tracks claim lifecycle |
| Vendor | featured | `Boolean` default false | Admin toggles for directory visibility |
| Review | vendorId + userId | `@@unique` | One review per user per vendor |
| VendorSummary | vendorId | `@unique` | One summary per vendor |
| Review | vendorId | `onDelete: Cascade` | Reviews deleted when vendor is deleted |
| Report | status | `ReportStatus` enum | Only valid statuses accepted |
| ContactMessage | status | `ContactMessageStatus` enum | Tracks support ticket resolution |
| BlacklistedToken | tokenHash | `@unique` | Prevent duplicate token revocations |
| BlacklistedToken | expiresAt | `@@index` | Efficient cleanup of expired tokens |
| RateLimit | key | `@id` (primary key) | One record per rate-limit key (e.g. IP address) |
| RateLimit | resetAt | `@@index` | Efficient cleanup of expired rate limit entries |

## Adding a New Field (Checklist)

1. Add the field to the correct model in `schema.prisma`
2. Run `npx prisma migrate dev --name add_[field]_to_[model]`
3. Run `npx prisma generate`
4. Update the corresponding TypeScript type in `server/src/types/`
5. Update any service that queries this model to include or exclude the new field as appropriate
6. Never return sensitive fields (e.g. `passwordHash`) in select queries
